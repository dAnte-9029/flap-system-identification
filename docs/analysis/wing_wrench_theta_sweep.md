# Wing-Only DeLaurier Wrench Theta Sweep

## Comparison objective and physical scope

This workflow evaluates a **wing-only DeLaurier baseline** on canonical real-flight rows and compares it with the **total reconstructed effective-wrench label** (`fx_b`, `fy_b`, `fz_b`, `mx_b`, `my_b`, `mz_b`). The label is the reconstructed whole-aircraft effective external wrench. It can contain wing, tail, fuselage, disturbance, left/right asymmetry, and reconstruction-error contributions, so it is not an isolated wing ground truth.

The sweep is a model sensitivity analysis. It does not identify real wing twist, fit `theta_tip`, train a corrected model, change the tail model, or regenerate labels.

## Isaac source provenance

The pure calculation is synchronized from `dAnte-9029/IsaacLab`, branch `flapping_rl`, frozen commit `3b5d4ec1d28f1384cf042402992ad7ea59995f49`. Runtime code in this repository is pure NumPy/Pandas and does not import IsaacLab, Isaac Sim, PhysX, or Torch. The numerical fixture in `tests/fixtures/isaaclab_delaurier_wing_wrench_3b5d4ec.json` was generated by executing the frozen Torch source directly from `git show` in an isolated module namespace. The float64 offline result is checked against strip components, one-wing Wang wrench, left/right body wrench, and total body-CG wrench with a maximum allowed absolute error of `1e-10`.

The frozen geometry is stored at `metadata/aircraft/flapper_01/wing_geometry_isaaclab_3b5d4ec.csv`; its hash is saved in every run manifest.

## Input contract

The primary attitude-aware run evaluates each canonical row using:

- `mechanical_phase_rad`;
- `flap_frequency_hz`;
- `vehicle_attitude.q[0..3]` (PX4 `wxyz`, body FRD to NED);
- `vehicle_local_position.vx/vy/vz` in NED;
- `wind.windspeed_north/east`, with vertical wind explicitly set to zero;
- `vehicle_air_data.rho`;
- `label_valid`, `phase_valid`, `cycle_valid`, `cycle_id`;
- `log_id`, `segment_id`, `time_s`, and the six effective-wrench labels.

The air-relative velocity is reconstructed in NED as ground velocity minus wind and rotated into body FRD with the measured quaternion. The frozen IsaacLab section contract is then applied as

\[
U=\max(u_b,0.5\ \mathrm{m/s}),\qquad
\theta_a=\operatorname{atan2}(w_b,u_b).
\]

The lateral component is retained as the diagnostic sideslip angle

\[
\beta=\operatorname{atan2}\!\left(v_b,\sqrt{u_b^2+w_b^2}\right),
\]

but is not consumed by the frozen two-dimensional strip equations. Logged true airspeed is saved only as a diagnostic in this mode. The isolated compatibility mode `legacy_scalar_true_airspeed` retains the original scalar true-airspeed plus `pitch_filtered` behavior; it was used only to quantify the effect of correcting the airflow contract.

The main phase-derivative mode is

\[
\dot\phi=2\pi f,\qquad \ddot\phi=0,
\]

which treats the current frequency as constant during a physics sample. Phase acceleration from a frequency derivative is not included in the main sweep. Horizontal wind is used, vertical wind is unavailable and set to zero, and sideslip is derived but not applied to the two-dimensional section model. Density comes only from the raw same-row `vehicle_air_data.rho`.

Each selected `(log_id, segment_id)` is evaluated in full before the output is cropped to requested windows. No window-local phase initialization, interpolation, derivative, filter, or physics state is used.

## Frame and reference contract

Canonical input and output use body FRD: `+x` forward, `+y` right, and `+z` down. The internal DeLaurier/Wang section frame is right-handed: `+x` root-to-tip along the pitching axis, `+y` surface-normal, and `+z` chordwise toward the leading edge. Canonical FRD airflow is passed through without a second sign flip. The frozen FLU wing-link geometry receives exactly one `diag(1,-1,-1)` conversion at the simulation-boundary equivalent.

Force is transformed as a polar vector. Moment is transformed as an axial vector, including determinant parity for the reflected right-wing mapping. No empirical right-wing theta sign is added.

The adapter parses the supplied YAML, requires body `FRD` plus IMU body/CG reference origins, and reads the stroke amplitude and real CG rather than treating the metadata as a hash-only input. The canonical body origin is the IMU origin. Current metadata gives the real-aircraft CG relative to the IMU as `[-0.12154, 0.00541, -0.04298] m`. Frozen wing roots converted to body FRD and expressed relative to the IMU are left `[0, -0.056, -0.030] m` and right `[0, +0.056, -0.030] m`. Each one-wing moment is first integrated about the wing-root pitching-axis origin and then translated to the real-aircraft CG:

\[
M_G=M_O+(p_O-p_G)\times F.
\]

The frozen Isaac base COM and the real metadata CG use consistent physical locations after accounting for the metadata IMU offset; the run manifest nevertheless records geometry and aircraft-metadata hashes separately so they cannot be silently mixed.

## Phase and dynamic twist

Metadata defines canonical physical motion as `q=A sin(phi_C)`, with `phi_C=0` at neutral wing starting upstroke. Frozen Isaac uses `q=Gamma cos(phi_D)`. The explicit mapping

\[
\phi_D=\operatorname{wrap}(\phi_C-\pi/2)
\]

preserves pose, motion direction, and derivative sign.

For each strip center `y_i`, the prescribed pitch is

\[
\delta\theta_i=-\theta_{\mathrm{tip}}\frac{y_i}{R}\sin\phi_D,
\]

\[
\delta\dot\theta_i=-\theta_{\mathrm{tip}}\frac{y_i}{R}\cos\phi_D\dot\phi_D,
\]

\[
\delta\ddot\theta_i=\theta_{\mathrm{tip}}\frac{y_i}{R}
\left(\sin\phi_D\dot\phi_D^2-\cos\phi_D\ddot\phi_D\right),
\qquad
\theta_i=\bar\theta_i+\delta\theta_i.
\]

`theta_tip` is the amplitude at the theoretical geometric tip `y=R`, not at the outermost strip center. The default sweep is `0, 5, 10, 15 deg`. The `0 deg` enabled mode is tested to be exactly identical to disabled dynamic twist.

## Window-selection rule

If no CSV manifest is supplied, the script builds deterministic 4 s candidates on a 2 s grid inside a single log and segment. A candidate must be fully label-valid, phase-valid, cycle-valid, airborne, finite in all physics inputs and labels, contain at least three cycle IDs, and have no time gap larger than eight times its median positive sample interval. Selection never reads baseline residuals.

Candidates are ranked for a stable straight window, low and high airspeed within the better half of stability scores, and windows nearest the 75th percentile of pitch-rate and yaw-rate RMS for mild pitch and turn coverage. Selected windows may not overlap by more than 25% inside the same segment. The resulting `selected_windows.csv` records all selection features and the rule.

## Metrics and bandwidth

For each window, theta value, and target, the workflow saves RMSE, MAE, mean bias (`baseline - label`), Pearson correlation, label standard deviation, baseline standard deviation, amplitude ratio, and valid sample count. A correlation involving a constant channel is saved as `NaN` with a reason, not zero. The same metrics are computed from per-cycle means. Phase folding defaults to 72 bins and saves mean, standard deviation, and count for both the label and baseline in all six axes.

The selected dataset reconstructs force and moment from kinematic derivatives smoothed by a 0.03 s, order-3 centered Savitzky-Golay operator applied within complete log/segment groups (`scipy.signal.savgol_filter`, `mode="interp"`). It has no intentional causal phase delay in the segment interior, but its derivative response and edge treatment do not define a single verified post-filter that maps raw aerodynamic wrench to these inverse-dynamics labels. Therefore the workflow always saves `baseline_raw`, does not claim a `baseline_label_bandwidth` result, and provides per-cycle means as a lower-bandwidth diagnostic. No arbitrary cutoff or time shift is applied.

## 2026-07-14 sensitivity result

Across the five windows, increasing `theta_tip` from 0 to 15 deg reduced the wing-only baseline standard deviation from `2.54` to `1.69 N` for Fx, `18.92` to `15.18 N` for Fz, and `1.29` to `1.05 N m` for My. Corresponding phase-folded peak-to-peak values decreased from `6.75` to `4.16 N`, `54.64` to `43.55 N`, and `4.12` to `3.25 N m`. The dominant extrema remained in nearly the same phase bins; the larger amplitude changes are more important than the small bin-level extrema shifts.

The mean of the per-cycle baseline values decreased from `3.16` to `1.74 N` for Fx and from `2.00` to `1.80 N` for Fz. The per-cycle My mean remained nearly unchanged (`-0.0950` to `-0.0954 N m`), while its within-cycle amplitude contracted. Fy remained identically zero because the baseline is symmetric. Mx and Mz are nonzero only through geometry/reference effects in the otherwise symmetric construction; their raw standard deviations decreased from `0.102` to `0.082 N m` and `0.0138` to `0.0092 N m`.

For the stable w01 component breakdown, the Fz RMS contribution from `dN_c` decreased from `20.25` to `15.88 N`; `dN_a` decreased from `5.73` to `4.30 N`. The chordwise Fx RMS contribution decreased from `3.77` to `1.95 N`. The My `r cross F` RMS contribution decreased from `1.36` to `1.03 N m`. The `dM_a` My RMS contribution grew from zero to `0.095 N m`, but remained much smaller than the force-arm term. Thus the observed My sensitivity is primarily normal/chordwise force redistribution through the wing/CG lever arms, with a smaller direct apparent-mass couple contribution.

The raw sample metrics against the whole-aircraft labels change with theta, but they must not be interpreted as twist identification. For example, the five-window mean raw Fz RMSE decreases from `14.83` to `12.28 N` and My RMSE from `1.34` to `1.14 N m`, while Fx RMSE is non-monotonic (`4.58`, `4.45`, `4.40`, `4.41 N`). These comparisons mix the wing-only model with total-label effects and do not establish a physically measured twist amplitude.

## Attitude-aware airflow rerun

The same five windows and theta values were rerun with `attitude_ground_wind_3d`. Across their theta-zero rows, reconstructed body-forward speed averaged `7.84 m/s`, body-down airflow averaged `2.66 m/s`, angle of attack averaged `18.98 deg`, and sideslip averaged `-2.16 deg`. Window-mean angle of attack ranged from `15.15` to `22.31 deg`; the mild-turn window had mean sideslip `-11.96 deg`.

This removes the artificial near-half-cycle antisymmetry in Fz produced by horizontal scalar airflow. In the stable `w01` window, the theta-zero baseline Fz mean changed from `-0.03 N` to `-14.85 N`; its minimum/maximum changed from approximately `-29.48/+30.20 N` to `-43.50/+10.03 N`. The normalized half-cycle antisymmetry error increased from `0.060` to `1.298`, confirming that the new Fz waveform contains a strong attitude-dependent mean and asymmetric half strokes.

At theta zero, the five-window mean raw metrics changed as follows:

| Target | Legacy RMSE | Attitude-aware RMSE | Legacy correlation | Attitude-aware correlation |
|---|---:|---:|---:|---:|
| Fx | 4.58 N | 3.86 N | 0.115 | 0.683 |
| Fz | 14.83 N | 10.18 N | 0.947 | 0.947 |
| My | 1.34 N m | 1.59 N m | 0.146 | 0.146 |

For the attitude-aware sweep, five-window mean RMSE at `0/5/10/15 deg` was `3.86/3.58/3.33/3.13 N` for Fx, `10.18/9.54/8.91/8.31 N` for Fz, and `1.59/1.56/1.52/1.49 N m` for My. These trends describe model sensitivity only. In particular, the force improvement and the theta-dependent My change do not identify the real twist because the comparison label still contains all aircraft loads.

## Limitations

The baseline contains wings only, while the labels represent total reconstructed effective wrench. The sweep is not a twist identification experiment, and a lower RMSE at one theta does not imply that value is the real twist. No direct twist measurement is available. Passive aeroelastic twist and structural dynamics are not modeled. Tail and fuselage loads are not included. Horizontal wind is used, but vertical wind is unavailable and assumed zero. Sideslip is diagnosed but not represented by the frozen two-dimensional strip model. Local flow from body angular velocity crossed with wing position is not added because it is absent from the frozen IsaacLab free-flight input contract. Symmetric baseline errors in Fy, Mx, and Mz can reflect real left/right asymmetry, tail/fuselage loads, disturbances, and label reconstruction as well as wing-model limitations. Baseline and label bandwidths are not exactly matched. No corrected model is trained or evaluated.

## Reproduction

```bash
MPLCONFIGDIR=/tmp/mplconfig XDG_CACHE_HOME=/tmp/xdg-cache \
python scripts/analyze_wing_wrench_theta_sweep.py \
  --split-root dataset/canonical_v0.2_training_ready_split_measured_massprops_ratio8_sg0p03_v1 \
  --partition test \
  --aircraft-metadata metadata/aircraft/flapper_01/aircraft_metadata.yaml \
  --theta-tip-deg 0 5 10 15 \
  --window-manifest docs/analysis/manifests/wing_wrench_theta_sweep_test_windows_20260714.csv \
  --phase-bins 72 \
  --airflow-mode attitude_ground_wind_3d \
  --legacy-comparison-aligned outputs/wing_wrench_theta_sweep/20260714_test_partition_frozen_3b5d4ec/aligned_predictions.parquet \
  --output-dir outputs/wing_wrench_theta_sweep/20260714_test_partition_attitude_airflow_frozen_3b5d4ec
```

For a fixed window manifest, add `--window-manifest path/to/windows.csv`. Arbitrary sweep lists are accepted after `--theta-tip-deg`.
